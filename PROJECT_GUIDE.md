# FPAMS Project Guide

## Objective

Develop a production-ready Faculty Performance Appraisal Management
System.

## Completed

-   Authentication & JWT
-   Global Exception Middleware
-   Department Master
-   Designation Master
-   Academic Year Master
-   User Management Backend
-   React Frontend Foundation

1.  Department Frontend CRUD
2.  Designation Frontend
3.  Academic Year Frontend
4.  Faculty Profile
5.  Appraisal Engine
6.  HOD Workflow
7.  Principal Workflow
8.  IQAC/APEC Workflow
9.  Reports
10. Notifications
11. File Uploads

## Development Rules

-   Zero build errors before proceeding.
-   Build and test after each sprint.
-   Use reusable components.
-   Follow Clean Architecture.
